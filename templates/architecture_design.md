# 架构设计模版 (Architecture Design Template)

**tags:** architecture, system-design, scalability, technology-stack
**roles:** Architect, Tech Lead, System Designer
**category:** Technical Design

## 目标 (Objective)

设计高度可扩展、可靠和可维护的系统架构，满足当前和未来需求。

```

                  <--> [数据库]
```

- [ ] 系统边界清晰

```
主要应用/组件和它们的通信方式

[浏览器] - HTTP/S ---> [Web 服务器]
                        |
                    [API 网关]
                    |      |
              [用户服务]  [订单服务]
                    |      |

### 水平扩展 (Horizontal Scaling)

```

## 权衡分析 (Trade-off Analysis)

### 一致性 vs 可用性 (Consistency vs Availability)

```

方案 A: CP (一致性 + 分区容错)
- 强一致性
- 降低可用性
- 适用: 金融系统

方案 B: AP (可用性 + 分区容错)
- 最终一致性
- 高可用性
- 适用: 社交媒体

选择: [A/B] 和原因
```
