# 研究员 主动思维模板 (Researcher Active Thinking Template)

## 使命 (Mission)

走在趋势前面、提供可行的洞见、减少知识差距

**Mission:** Stay ahead of trends, provide actionable insights, reduce knowledge gaps

---

## 思维方向 (Thinking Directions)

- **新兴技术 (Emerging Technologies)** - 有哪些可能影响我们的新技术？
- **竞争对手动向 (Competitor Movements)** - 竞争对手在做什么？他们的策略是什么？
- **行业变化 (Industry Shifts)** - 市场和行业如何变化？有哪些新趋势？
- **用户行为变化 (User Behavior Changes)** - 用户需求和偏好如何演变？
- **监管变化 (Regulatory Changes)** - 监管环境如何变化？有什么风险？
- **团队知识差距 (Team Knowledge Gaps)** - 团队在哪些领域缺乏知识？

---

## 思维模板 (Thinking Template)

### 1. 研究景观 (Research Landscape)

当前的技术和市场景观是什么？

- [ ] **技术生态 (Technology Ecosystem)**
  - [ ] 我们领域的主要技术趋势是什么？
  - [ ] 是否有新兴的编程语言、框架或工具？
  - [ ] 云、AI、区块链等技术的最新发展如何？
  - [ ] 哪些技术正在被行业领导者采用？
  - [ ] 哪些旧技术正在被淘汰？

- [ ] **竞争格局 (Competitive Landscape)**
  - [ ] 主要竞争对手有哪些？
  - [ ] 他们的产品功能和优势是什么？
  - [ ] 他们的定价策略和市场定位？
  - [ ] 有哪些新的或隐形竞争对手？
  - [ ] 市场份额如何分布？

- [ ] **行业趋势 (Industry Trends)**
  - [ ] 用户的最大需求或痛点是什么？
  - [ ] 有哪些新兴的细分市场或应用场景？
  - [ ] 监管环境如何变化？
  - [ ] 行业的整合或颠覆是否发生？
  - [ ] 行业增长预测是什么？

- [ ] **用户和市场动态 (User & Market Dynamics)**
  - [ ] 用户行为如何变化？
  - [ ] 用户优先级或偏好是否在变化？
  - [ ] 是否出现了新的用户群体或使用场景？
  - [ ] 用户满意度或采用率的趋势如何？
  - [ ] 是否存在用户需求与现有产品之间的差距？

- [ ] **经济和宏观因素 (Economic & Macro Factors)**
  - [ ] 经济周期对我们领域的影响如何？
  - [ ] 是否存在通胀、汇率或其他宏观风险？
  - [ ] 投资趋势如何？VC / 股权融资是否充足？
  - [ ] 人才市场如何？是否存在招聘挑战？

**Landscape Assessment:**
- [ ] 当前景观中最重要的 3-5 个变化是什么？
- [ ] 这些变化如何影响我们的业务或技术？
- [ ] 我们的相对竞争地位如何？

---

### 2. 关键发现 (Key Findings)

通过研究发现了哪些关键洞见？

- [ ] **技术洞见 (Technology Insights)**
  - [ ] 有哪些有前景的新技术？
  - [ ] 这些技术如何应用于我们的问题？
  - [ ] 采用这些技术的成本和收益是什么？
  - [ ] 我们竞争对手是否已在使用这些技术？
  - [ ] 采用的时机是否成熟？

- [ ] **竞争洞见 (Competitive Insights)**
  - [ ] 竞争对手的优势和劣势是什么？
  - [ ] 他们的产品路线图或战略方向是什么？
  - [ ] 我们如何与他们比较？
  - [ ] 我们可以从他们那里学到什么？
  - [ ] 我们的差异化优势是什么？

- [ ] **市场洞见 (Market Insights)**
  - [ ] 新的市场机会或细分市场是什么？
  - [ ] 用户的未被满足的需求是什么？
  - [ ] 进入新市场或服务新用户的成本是什么？
  - [ ] 这些市场的增长潜力有多大？
  - [ ] 我们的进入策略应该是什么？

- [ ] **用户行为洞见 (User Behavior Insights)**
  - [ ] 用户如何使用产品正在发生什么变化？
  - [ ] 新兴的使用场景或用户需求？
  - [ ] 用户放弃或转向竞争对手的原因？
  - [ ] 不同用户群体的优先级或期望如何不同？
  - [ ] 我们应该如何调整产品或营销？

- [ ] **监管和合规洞见 (Regulatory & Compliance Insights)**
  - [ ] 即将到来的监管变化是什么？
  - [ ] 这对我们的业务或产品有什么影响？
  - [ ] 我们应该如何准备？
  - [ ] 竞争对手如何应对这些变化？
  - [ ] 是否有合规的机会来获得竞争优势？

**Finding Validation:**
- [ ] 这些发现基于什么证据？（数据、访谈、文献等）
- [ ] 是否存在相互矛盾的发现或不确定性？
- [ ] 这些发现的时效性如何？

---

### 3. 趋势分析 (Trend Analysis)

识别出了哪些关键趋势？它们的含义是什么？

- [ ] **长期趋势 (Long-Term Trends)**
  - [ ] 过去 2-3 年的主要趋势是什么？
  - [ ] 这些趋势可能持续多长时间？
  - [ ] 未来 3-5 年的预期方向是什么？
  - [ ] 这些趋势对我们的业务意味着什么？
  - [ ] 我们应该如何定位自己？

- [ ] **短期动向 (Near-Term Dynamics)**
  - [ ] 未来 6-12 个月会发生什么变化？
  - [ ] 有哪些即将到来的关键事件或发布？
  - [ ] 我们应该立即做什么来保持竞争力？
  - [ ] 有哪些快速变化的领域需要密切监控？

- [ ] **新兴机会 (Emerging Opportunities)**
  - [ ] 哪些新兴趋势为我们创造了机会？
  - [ ] 我们可以如何利用这些机会？
  - [ ] 窗口期有多长？
  - [ ] 我们需要什么才能利用这些机会？
  - [ ] 首先应该追求哪些机会？

- [ ] **潜在威胁 (Potential Threats)**
  - [ ] 哪些趋势可能威胁我们的业务？
  - [ ] 这些威胁有多真实和紧迫？
  - [ ] 我们应该如何防御？
  - [ ] 我们应该监控什么信号来预警？
  - [ ] 应该优先解决哪些威胁？

**Trend Implications:**
- [ ] 如果这些趋势继续，未来 18 个月会是什么样的？
- [ ] 哪个单一趋势影响最大？
- [ ] 我们的战略应该如何调整？

---

### 4. 可行建议 (Actionable Recommendations)

基于研究，应该做什么？

- [ ] **产品建议 (Product Recommendations)**
  - [ ] 产品应该如何演变来适应趋势？
  - [ ] 应该构建哪些新功能？
  - [ ] 应该淘汰或弃用哪些功能？
  - [ ] 应该调整的定价或包装是什么？
  - [ ] 应该何时实施这些变化？

- [ ] **技术建议 (Technology Recommendations)**
  - [ ] 应该采用哪些新技术？
  - [ ] 应该淘汰哪些技术？
  - [ ] 应该进行什么技术投资？
  - [ ] 应该建立什么能力？
  - [ ] 实施时间表应该是什么？

- [ ] **业务战略建议 (Business Strategy Recommendations)**
  - [ ] 应该进入哪些新市场或细分市场？
  - [ ] 应该采购哪些合作伙伴或技术？
  - [ ] 应该如何调整营销或销售？
  - [ ] 应该如何对标竞争对手？
  - [ ] 应该何时采取行动？

- [ ] **组织建议 (Organization Recommendations)**
  - [ ] 应该建立什么新团队或能力？
  - [ ] 应该聘请什么专业人才？
  - [ ] 应该投资什么培训或发展？
  - [ ] 应该如何组织团队来利用机会？

**Recommendation Prioritization:**
- [ ] 按影响力和可行性排序建议
- [ ] 对于每个建议，确定 6 个月内的行动项

---

### 5. 知识差距 (Knowledge Gaps)

在当前的理解中存在哪些差距？

- [ ] **研究覆盖 (Research Coverage)**
  - [ ] 研究中缺少哪些关键区域？
  - [ ] 是否有我们没有足够信息的主题？
  - [ ] 我们对竞争对手的了解是否充分？
  - [ ] 我们对用户需求的了解是否充分？
  - [ ] 我们是否遗漏了任何关键的市场细分？

- [ ] **团队知识 (Team Knowledge)**
  - [ ] 团队是否理解这些研究发现？
  - [ ] 团队中是否有针对关键领域的深度知识？
  - [ ] 是否需要外部专家或顾问？
  - [ ] 应该进行什么培训？
  - [ ] 应该记录什么知识？

- [ ] **数据和验证 (Data & Validation)**
  - [ ] 我们是否有足够的数据来支持这些发现？
  - [ ] 是否应该进行用户研究或调查？
  - [ ] 是否应该进行竞争分析或基准测试？
  - [ ] 是否应该进行试点或概念验证？
  - [ ] 应该优先进行什么验证？

**Gap Closure Plan:**
- [ ] 对于每个关键差距，如何填补它？
- [ ] 需要什么资源或投入？
- [ ] 时间表应该是什么？

### 6. 传播计划 (Dissemination Plan)

如何与组织分享研究发现？

- [ ] **关键利益相关者 (Key Stakeholders)**
  - [ ] CEO / 创始人应该知道什么？
  - [ ] 产品团队应该知道什么？
  - [ ] 工程团队应该知道什么？
  - [ ] 营销团队应该知道什么？
  - [ ] 其他部门应该知道什么？

- [ ] **通信形式 (Communication Format)**
  - [ ] 应该创建什么文档或报告？
  - [ ] 应该进行什么演讲或讨论？
  - [ ] 应该创建什么决策支持材料？
  - [ ] 应该如何更新内部知识库？
  - [ ] 应该进行什么后续行动？

- [ ] **时间表 (Timeline)**
  - [ ] 何时应该分享初步发现？
  - [ ] 何时应该进行全面简报？
  - [ ] 应该多久更新一次发现？
  - [ ] 应该如何处理新信息或变化？

- [ ] **持续参与 (Ongoing Engagement)**
  - [ ] 应该建立什么论坛或讨论？
  - [ ] 应该收集什么反馈？
  - [ ] 应该如何与内部专家协作？
  - [ ] 应该建立什么监控或预警？

**Dissemination Effectiveness:**
- [ ] 如何衡量分享内容的理解和影响？
- [ ] 应该如何调整沟通方式？

### 7. 自我反思 (Self-Reflection)

- [ ] **研究质量 (Quality of Research)**
  - [ ] 我是否进行了充分的研究？
  - [ ] 我的信息来源是否可靠？
  - [ ] 我是否有确认偏见或遗漏了相反的证据？
  - [ ] 我的分析是否有逻辑上的缺陷？
  - [ ] 我的结论是否过度推断了数据？

- [ ] **研究有效性 (Research Validity)**
  - [ ] 我的发现与我的行业知识一致吗？
  - [ ] 是否有与我的结论不一致的数据点？
  - [ ] 我的时间范围是否合适？
  - [ ] 是否存在我没有考虑的外部因素？
  - [ ] 我是否充分验证了我的假设？

- [ ] **影响和相关性 (Impact & Relevance)**
  - [ ] 我的发现对组织来说是相关的吗？
  - [ ] 我的建议是否可行和有行动力？
  - [ ] 我是否充分强调了关键问题？
  - [ ] 我是否忽视了任何高优先级的领域？
  - [ ] 我的沟通是否有效？

- [ ] **学习和改进 (Learning & Improvement)**
  - [ ] 通过这个研究我学到了什么？
  - [ ] 我的研究方法是否可以改进？
  - [ ] 是否有我应该发展的新技能？
  - [ ] 是否有我应该跟踪的新信息来源？

- [ ] **下一步 (Next Steps)**
  - [ ] 应该优先进行什么后续研究？
  - [ ] 应该建立什么持续的监控？
  - [ ] 应该何时更新这些发现？
  - [ ] 应该与谁讨论建议？

## 检查清单 (Checklist)

在完成这个思维练习后，确认：

- [ ] 识别了 3-5 个对业务或技术最重要的趋势
- [ ] 识别了 3-5 个具体可行的建议
- [ ] 评估了关键的市场、技术和竞争机会
- [ ] 确定了关键的知识差距
- [ ] 制定了与组织分享发现的计划
- [ ] 承认了研究中的不确定性和假设
- [ ] 规划了后续研究或验证

---

## 实施与跟踪 (Implementation & Tracking)

- **频率 (Frequency)** - 每月进行一次深度研究，每周进行一次洞见汇总和趋势监控
- **沟通 (Communication)** - 定期向关键利益相关者分享发现，根据优先级定期更新
- **监控 (Monitoring)** - 持续关注关键趋势和竞争对手动向
- **反馈 (Feedback)** - 从团队收集反馈，验证假设，改进未来研究
