# 设计师 主动思维模板 (Designer Active Thinking Template)

## 使命 (Mission)

用户体验卓越、设计一致性、可访问性

**Mission:** User experience excellence, design consistency, and accessibility

---

## 思维方向 (Thinking Directions)

- **可用性痛点 (Usability Pain Points)** - 用户在哪里遇到困难？
- **设计不一致 (Design Inconsistencies)** - 产品中的设计偏离或不一致在哪里？
- **可访问性缺陷 (Accessibility Gaps)** - 有多少用户因为可访问性问题而被排除？
- **用户反馈模式 (User Feedback Patterns)** - 用户反复提出什么反馈？
- **竞争 UX 分析 (Competitive UX Analysis)** - 竞争对手如何处理类似的问题？
- **设计系统漂移 (Design System Drift)** - 产品是否偏离了设计系统？

---

## 思维模板 (Thinking Template)

### 1. UX 状态评估 (UX Status Assessment)

当前的用户体验状况如何？

- [ ] **用户满意度 (User Satisfaction)**
  - [ ] 当前的用户满意度评分或 NPS？
  - [ ] 用户最满意什么？
  - [ ] 用户最不满意什么？
  - [ ] 满意度趋势如何（改进、稳定、恶化）？
  - [ ] 不同用户段的满意度如何不同？

- [ ] **可用性指标 (Usability Metrics)**
  - [ ] 关键任务的完成率如何？
  - [ ] 用户完成任务需要多长时间？
  - [ ] 用户进行的错误操作有多少？
  - [ ] 用户放弃任务的频率？
  - [ ] 用户对功能可发现性的感受？

- [ ] **设计一致性 (Design Consistency)**
  - [ ] 产品中的设计是否一致？
  - [ ] 是否遵循了设计系统？
  - [ ] 不同平台或页面之间的一致性如何？
  - [ ] 是否存在明显的风格或交互模式偏离？
  - [ ] 设计系统是否最新且完整？

- [ ] **可访问性评估 (Accessibility Assessment)**
  - [ ] WCAG 符合程度（A、AA、AAA）？
  - [ ] 是否支持屏幕阅读器？
  - [ ] 是否支持键盘导航？
  - [ ] 色彩对比是否充分？
  - [ ] 是否有关键的可访问性缺陷？

- [ ] **用户反馈和研究 (User Feedback & Research)**
  - [ ] 最近的用户研究发现是什么？
  - [ ] 用户反馈中最常见的主题是什么？
  - [ ] 用户是否提出了新的需求或痛点？
  - [ ] 是否进行了可用性测试？

**UX Health Score:**
- [ ] 给整体 UX 健康度评分 (1-10)
- [ ] 最大的 UX 问题是什么？
- [ ] 与竞争对手相比如何？

---

### 2. 痛点和机会 (Pain Points & Opportunities)

用户体验中存在哪些主要痛点？

- [ ] **关键用户旅程痛点 (Critical User Journey Pain Points)**
  - [ ] 用户在入门过程中遇到什么困难？
  - [ ] 用户在核心任务中遇到什么困难？
  - [ ] 用户在完成交互或购买时遇到什么困难？
  - [ ] 用户在需要帮助时遇到什么困难？
  - [ ] 哪个阶段的放弃率最高？

- [ ] **界面和交互问题 (Interface & Interaction Issues)**
  - [ ] 有哪些令人困惑的界面元素？
  - [ ] 导航是否直观？
  - [ ] 是否存在隐藏的或难以发现的功能？
  - [ ] 表单是否容易填写？
  - [ ] 错误消息是否有帮助？

- [ ] **移动和响应式问题 (Mobile & Responsiveness Issues)**
  - [ ] 移动体验如何？
  - [ ] 不同屏幕大小的体验是否一致？
  - [ ] 触摸目标大小是否合适？
  - [ ] 加载时间是否可接受？
  - [ ] 移动特定的功能是否缺失？

- [ ] **可访问性问题 (Accessibility Issues)**
  - [ ] 哪些用户因为可访问性问题而被排除？
  - [ ] 最常见的可访问性障碍是什么？
  - [ ] 屏幕阅读器用户遇到什么困难？
  - [ ] 键盘用户遇到什么困难？
  - [ ] 有视觉或听力障碍的用户遇到什么困难？

- [ ] **性能感知 (Performance Perception)**
  - [ ] 用户对加载速度的看法？
  - [ ] 是否有明显的性能问题影响 UX？
  - [ ] 不同网络条件下的体验如何？
  - [ ] 离线或低连接情况下的支持？

- [ ] **内容和信息架构 (Content & Information Architecture)**
  - [ ] 内容是否清晰易懂？
  - [ ] 信息是否有组织？
  - [ ] 用户是否容易找到他们需要的？
  - [ ] 是否存在信息过载？
  - [ ] 帮助和文档是否充分？

**Problem Prioritization:**
- [ ] 按影响用户数量排序
- [ ] 按对业务指标的影响排序
- [ ] 确定 3-5 个优先级最高的痛点

### 3. 设计改进计划 (Design Improvement Plan)

如何改进用户体验？

- [ ] **用户研究和验证 (User Research & Validation)**
  - [ ] 应该进行什么用户研究来验证痛点？
  - [ ] 应该进行可用性测试吗？
  - [ ] 应该收集什么用户反馈？
  - [ ] 应该进行什么 A/B 测试？

- [ ] **设计改进 (Design Improvements)**
  - [ ] 针对最关键的痛点的具体改进是什么？
  - [ ] 应该重新设计哪些界面或流程？
  - [ ] 应该添加或删除什么功能？
  - [ ] 应该如何改进信息架构？
  - [ ] 应该如何改进导航？

- [ ] **可访问性改进 (Accessibility Improvements)**
  - [ ] 应该如何提高 WCAG 符合程度？
  - [ ] 应该改进什么屏幕阅读器支持？
  - [ ] 应该改进什么键盘导航？
  - [ ] 应该改进什么色彩对比？
  - [ ] 应该改进什么动画或闪烁问题？

- [ ] **设计系统改进 (Design System Improvements)**
  - [ ] 应该如何更新设计系统？
  - [ ] 应该添加什么新组件或模式？
  - [ ] 应该标准化什么现有的不一致？
  - [ ] 应该如何推广设计系统？
  - [ ] 应该如何确保设计系统的采用？

- [ ] **跨平台一致性 (Cross-Platform Consistency)**
  - [ ] 不同平台之间应该如何一致？
  - [ ] 应该如何适应不同的平台约束？
  - [ ] 应该如何处理平台特定的功能？

**Improvement Prioritization:**
- [ ] 按优先级排序（影响 x 可行性）
- [ ] 选择 3-5 个可以在下季度启动的改进

---

### 4. 用户测试计划 (User Testing Plan)

如何验证改进的有效性？

- [ ] **可用性测试 (Usability Testing)**
  - [ ] 应该测试哪些用户流或功能？
  - [ ] 应该招募什么类型的用户？
  - [ ] 测试方法应该是什么？（调查、面对面、远程）
  - [ ] 应该测试多少用户？
  - [ ] 什么时候应该进行测试？

- [ ] **A/B 测试 (A/B Testing)**
  - [ ] 应该 A/B 测试什么设计变化？
  - [ ] 如何确定统计显著性？
  - [ ] 样本量应该是多少？
  - [ ] 测试应该运行多久？

- [ ] **可访问性测试 (Accessibility Testing)**
  - [ ] 应该进行什么自动化可访问性检查？
  - [ ] 应该进行什么手动可访问性审计？
  - [ ] 应该与真实用户进行可访问性测试吗？
  - [ ] 应该测试什么屏幕阅读器和浏览器组合？

- [ ] **性能感知测试 (Performance Perception Testing)**
  - [ ] 应该如何测试加载时间的感知？
  - [ ] 应该进行什么真实用户监控？

---

### 5. 设计系统更新 (Design System Updates)

设计系统应该如何演变？

- [ ] **组件库更新 (Component Library Updates)**
  - [ ] 应该添加什么新组件？
  - [ ] 应该更新或弃用什么组件？
  - [ ] 组件应该如何改进？
  - [ ] 应该如何文档化组件？

- [ ] **风格和主题 (Styles & Themes)**
  - [ ] 应该如何改进设计令牌（颜色、排版、间距）？
  - [ ] 应该支持什么主题（浅色、深色等）？
  - [ ] 应该如何处理国际化和本地化？

- [ ] **模式和最佳实践 (Patterns & Best Practices)**
  - [ ] 应该记录什么常见的 UX 模式？
  - [ ] 应该记录什么设计最佳实践？
  - [ ] 应该如何推广这些模式？
  - [ ] 应该如何处理例外或特殊情况？

- [ ] **工具和流程 (Tools & Processes)**
  - [ ] 应该使用什么设计工具？
  - [ ] 应该如何改进设计到开发的交接？
  - [ ] 应该如何保持设计系统与实现的同步？
  - [ ] 应该如何收集和处理设计反馈？

---

### 6. 性能指标 (Performance Metrics)

如何衡量 UX 改进的有效性？

- [ ] **用户满意度指标 (User Satisfaction Metrics)**
  - [ ] NPS 或 CSAT 的目标
  - [ ] 用户研究发现的改变
  - [ ] 定性反馈的改变

- [ ] **行为指标 (Behavioral Metrics)**
  - [ ] 关键任务完成率的改变
  - [ ] 用户流失或弃用率的改变
  - [ ] 功能采用率的改变
  - [ ] 用户参与度的改变

- [ ] **业务指标 (Business Metrics)**
  - [ ] 转化率的改变
  - [ ] 用户保留率的改变
  - [ ] 收入或 ARPU 的改变
  - [ ] 支持请求数量的改变

- [ ] **设计质量指标 (Design Quality Metrics)**
  - [ ] 设计系统采用率
  - [ ] 一致性得分
  - [ ] 可访问性符合程度
  - [ ] 性能指标

---

### 7. 自我反思 (Self-Reflection)

- [ ] **设计分析质量 (Quality of Design Analysis)**
  - [ ] 我是否充分考虑了所有用户段？
  - [ ] 我是否遗漏了任何关键的可用性问题？
  - [ ] 我的优先级是否基于真实的用户数据？
  - [ ] 我是否考虑了可访问性和包容性？

- [ ] **创意和创新 (Creativity & Innovation)**
  - [ ] 我的设计解决方案是否创新？
  - [ ] 我是否寻求了灵感或参考？
  - [ ] 我是否跳出了框框思考？
  - [ ] 我的设计是否与产品战略对齐？

- [ ] **用户同理心 (User Empathy)**
  - [ ] 我是否真正理解了用户的痛点？
  - [ ] 我的设计是否考虑了用户的背景和限制？
  - [ ] 我是否充分考虑了可访问性？
  - [ ] 我是否与真实用户交流过？

- [ ] **执行力 (Execution Reality)**
  - [ ] 我的设计是否现实可行？
  - [ ] 工程团队是否能实施我的设计？
  - [ ] 是否有技术限制我没有考虑？
  - [ ] 成本和资源是否充分？

- [ ] **学习和改进 (Learning & Improvement)**
  - [ ] 最近的设计尝试或失败中学到了什么？
  - [ ] 我如何改进我的设计流程？
  - [ ] 是否有新的设计趋势或技术我应该学习？
  - [ ] 我的设计技能在哪些方面最需要改进？

- [ ] **下一步 (Next Steps)**
  - [ ] 最优先的设计行动项是什么？
  - [ ] 应该与谁合作来实施这些改进？
  - [ ] 应该何时进行用户研究或测试？
  - [ ] 应该如何与工程和产品团队协调？

## 检查清单 (Checklist)

在完成这个思维练习后，确认：

- [ ] 进行了全面的用户体验评估
- [ ] 识别了 3-5 个关键的用户痛点
- [ ] 识别了 3-5 个改进机会
- [ ] 制定了改进的优先级
- [ ] 规划了用户研究或测试
- [ ] 评估了可访问性并制定了改进计划
- [ ] 规划了设计系统的更新
- [ ] 定义了成功的衡量标准

---

## 实施与跟踪 (Implementation & Tracking)

- **频率 (Frequency)** - 每两周进行一次用户体验审查，每月进行一次深度分析
- **沟通 (Communication)** - 定期与产品和工程团队分享设计洞见和改进
- **行动 (Action)** - 将改进转化为具体的设计任务或原型
- **追踪 (Tracking)** - 定期监控用户满意度、可用性和可访问性指标
