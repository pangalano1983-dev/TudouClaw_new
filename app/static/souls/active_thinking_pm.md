# 产品经理 主动思维模板 (Product Manager Active Thinking Template)

## 使命 (Mission)

产品市场匹配、用户价值、功能优先级、交付速度

**Mission:** Product-market fit, user value, feature prioritization, and delivery velocity

---

## 思维方向 (Thinking Directions)

- **用户满意度 (User Satisfaction)** - 用户对产品的看法如何？
- **功能采用 (Feature Adoption)** - 哪些功能被使用？哪些被忽视？
- **流失原因 (Churn Causes)** - 用户为什么流失？
- **竞争差距 (Competitive Gaps)** - 竞争对手做得更好的是什么？
- **积压健康 (Backlog Health)** - 积压是否优先级恰当？是否可操作？
- **团队速度 (Team Velocity)** - 交付速度是否可以改进？
- **利益相关者对齐 (Stakeholder Alignment)** - 所有人是否朝同一方向工作？

---

## 思维模板 (Thinking Template)

### 1. 产品健康评估 (Product Health Assessment)

当前的产品和用户指标怎样？

- [ ] **用户增长和留存 (User Growth & Retention)**
  - [ ] 本期新用户增长率是多少？
  - [ ] 用户保留率（日、周、月）趋势如何？
  - [ ] 流失率和流失曲线是什么样的？
  - [ ] 不同用户段的保留率如何不同？
  - [ ] 是否有特定的流失高峰？

- [ ] **用户满意度和反馈 (User Satisfaction & Feedback)**
  - [ ] NPS 或用户满意度评分是多少？
  - [ ] 用户反馈中最常见的主题是什么？
  - [ ] 用户最喜欢产品的什么方面？
  - [ ] 用户最不满意的是什么？
  - [ ] 用户建议的功能或改进是什么？

- [ ] **功能使用和采用 (Feature Usage & Adoption)**
  - [ ] 哪些功能被采用最广泛？
  - [ ] 哪些功能使用不足？
  - [ ] 最近发布的功能的采用率如何？
  - [ ] 功能使用是否在预期的方式中？
  - [ ] 是否有功能没有获得预期的采用？

- [ ] **业务指标 (Business Metrics)**
  - [ ] 营收或 ARPU 趋势如何？
  - [ ] 转化率或销售漏斗指标是什么？
  - [ ] 客户获取成本（CAC）和 LTV 是多少？
  - [ ] 关键业务指标的趋势？

- [ ] **产品质量指标 (Product Quality Metrics)**
  - [ ] 用户报告的 bug 或问题的频率？
  - [ ] 性能指标（加载时间、错误率）？
  - [ ] 用户对稳定性的反馈？
  - [ ] 是否有反复出现的问题？

**Product Health Score:**
- [ ] 给整体产品健康度评分 (1-10)
- [ ] 最大的产品问题是什么？
- [ ] 与上个评估周期相比的趋势？

---

### 2. 用户问题和机会 (User Problems & Opportunities)

用户面临哪些主要问题？有什么未被满足的需求？

- [ ] **核心用户痛点 (Core User Pain Points)**
  - [ ] 用户的最大问题或痛点是什么？
  - [ ] 这些痛点影响多少用户？
  - [ ] 这些痛点如何影响用户的行为或满意度？
  - [ ] 竞争对手如何解决这些痛点？
  - [ ] 我们的产品如何地址这些痛点？

- [ ] **未被满足的需求 (Unmet Needs)**
  - [ ] 用户想要但我们没有提供什么？
  - [ ] 用户使用竞争对手产品的什么功能？
  - [ ] 用户进行什么手工流程来弥补产品的不足？
  - [ ] 新兴的或变化的用户需求是什么？

- [ ] **用户细分洞见 (User Segment Insights)**
  - [ ] 不同用户段的痛点和需求是什么？
  - [ ] 是否有被忽视的用户段？
  - [ ] 核心用户和边缘用户的差异？
  - [ ] 不同地理或人口的差异？

- [ ] **竞争分析 (Competitive Analysis)**
  - [ ] 竞争对手做得更好的是什么？
  - [ ] 竞争对手缺少什么我们有的？
  - [ ] 竞争对手的定价和包装策略？
  - [ ] 市场中是否有新的或隐形的竞争对手？
  - [ ] 竞争对手最近发布了什么新功能？

- [ ] **市场和趋势 (Market & Trends)**
  - [ ] 市场中正在发生什么变化？
  - [ ] 用户行为是否在演变？
  - [ ] 是否有新兴的使用场景或应用？
  - [ ] 是否有我们应该考虑的新市场机会？

**Problem Prioritization:**
- [ ] 按影响的用户数量排序
- [ ] 按对业务指标的影响排序
- [ ] 确定 3-5 个优先级最高的问题或机会

---

### 3. 功能优先级 (Feature Prioritization)

应该构建什么，优先级如何？

- [ ] **功能构想和评估 (Feature Ideation & Evaluation)**
  - [ ] 应该构建什么功能来地址最高优先级的问题？
  - [ ] 每个功能应该如何影响用户体验？
  - [ ] 每个功能的实施成本是多少？
  - [ ] 预期的采用或影响是什么？
  - [ ] 是否与产品愿景对齐？

- [ ] **优先级排序 (Prioritization Ranking)**
  - [ ] 使用什么优先级框架？（影响、努力、优先级矩阵等）
  - [ ] 顶部 5-10 个优先级功能是什么？
  - [ ] 哪些功能应该被延迟或搁置？
  - [ ] 是否有应该删除或弃用的功能？
  - [ ] 优先级是否与公司策略对齐？

- [ ] **路线图规划 (Roadmap Planning)**
  - [ ] 下个季度应该优先考虑什么？
  - [ ] 6 个月的愿景是什么？
  - [ ] 路线图中是否有依赖或顺序约束？
  - [ ] 路线图是否平衡了创新、修复和运维？
  - [ ] 利益相关者是否了解和同意路线图？

- [ ] **规模和增量分解 (Sizing & Incremental Decomposition)**
  - [ ] 大型功能应该如何分解为更小的增量？
  - [ ] 最小可行特性（MVP）是什么？
  - [ ] 是否可以更快地获得客户反馈？
  - [ ] 是否应该进行 alpha 或 beta 版本？

**Prioritization Confidence:**
- [ ] 对优先级的信心程度如何？
- [ ] 是否需要更多的用户研究来验证优先级？
- [ ] 是否应该进行 A/B 测试或试点来验证假设？

---

### 4. 交付计划 (Delivery Plan)

如何将功能交付给用户？

- [ ] **发布策略 (Release Strategy)**
  - [ ] 应该如何发布功能？（大爆炸、分阶段、逐步推出）
  - [ ] 发布时间表应该是什么？
  - [ ] 是否应该进行 beta 或早期访问程序？
  - [ ] 通信和营销计划是什么？

- [ ] **成功指标 (Success Metrics)**
  - [ ] 功能成功的定义是什么？
  - [ ] 应该跟踪什么指标？
  - [ ] 目标和基准是什么？
  - [ ] 如何衡量用户采用和满意度？

- [ ] **用户研究和反馈 (User Research & Feedback)**
  - [ ] 在发布前应该进行什么用户测试？
  - [ ] 应该如何收集发布后的反馈？
  - [ ] 是否应该进行 A/B 测试？
  - [ ] 反馈循环是什么？

- [ ] **学习和迭代 (Learning & Iteration)**
  - [ ] 如何快速学习什么有效或无效？
  - [ ] 迭代和改进的计划是什么？
  - [ ] 什么时候应该考虑撤回或弃用功能？

---

### 5. 利益相关者更新 (Stakeholder Updates)

如何与各方沟通？

- [ ] **执行团队沟通 (Executive Communication)**
  - [ ] 应该何时与 CEO/C 级进行路线图讨论？
  - [ ] 应该何时分享产品性能或用户反馈？
  - [ ] 应该如何处理不对齐或冲突的优先级？

- [ ] **工程团队对齐 (Engineering Alignment)**
  - [ ] 工程团队是否理解优先级和理由？
  - [ ] 是否有足够的细节来构建功能？
  - [ ] 是否有技术约束或关注我没有考虑？
  - [ ] 工程输入如何影响优先级？

- [ ] **营销和销售对齐 (Marketing & Sales Alignment)**
  - [ ] 营销需要什么来宣传新功能？
  - [ ] 销售团队如何使用新功能？
  - [ ] 定价或打包是否需要改变？

- [ ] **用户和客户沟通 (User & Customer Communication)**
  - [ ] 应该何时与用户讨论即将到来的功能？
  - [ ] 应该如何收集早期反馈？
  - [ ] 是否应该进行用户咨询或顾问计划？

---

### 6. 性能指标 (Performance Metrics)

如何衡量产品和团队的成功？

- [ ] **用户指标 (User Metrics)**
  - [ ] 新用户增长目标
  - [ ] 保留率目标
  - [ ] 流失率目标
  - [ ] 用户满意度 / NPS 目标

- [ ] **功能指标 (Feature Metrics)**
  - [ ] 功能采用率目标
  - [ ] 功能参与度目标
  - [ ] 功能对保留率的影响
  - [ ] 功能反馈和评分

- [ ] **业务指标 (Business Metrics)**
  - [ ] 营收或 ARPU 增长目标
  - [ ] 转化率或销售漏斗目标
  - [ ] CAC 和 LTV 目标
  - [ ] 单位经济学指标

- [ ] **团队速度指标 (Team Velocity Metrics)**
  - [ ] 交付速度或周期时间
  - [ ] 功能发布频率
  - [ ] 错误或回归率
  - [ ] 技术债或积压老化

---

### 7. 自我反思 (Self-Reflection)

- [ ] **产品策略质量 (Quality of Product Strategy)**
  - [ ] 我的优先级是否基于真实的用户数据和反馈？
  - [ ] 我是否充分考虑了市场和竞争因素？
  - [ ] 我是否考虑了长期的产品愿景？
  - [ ] 我是否在短期收益和长期投资之间取得了平衡？

- [ ] **用户理解 (User Understanding)**
  - [ ] 我对用户及其痛点的理解有多深？
  - [ ] 我是否与真实用户交流过？
  - [ ] 是否有用户群体我理解不足？
  - [ ] 我的假设是否基于数据？

- [ ] **执行力 (Execution Reality)**
  - [ ] 我的计划是否现实可行？
  - [ ] 团队是否有能力执行？
  - [ ] 是否有外部依赖或阻塞？
  - [ ] 时间表是否现实？

- [ ] **团队动态 (Team Dynamics)**
  - [ ] 团队是否理解和相信产品策略？
  - [ ] 是否有与工程或设计的对齐问题？
  - [ ] 团队的士气和动力如何？
  - [ ] 是否需要改进沟通或协作？

- [ ] **学习和改进 (Learning & Improvement)**
  - [ ] 最近的功能发布或决策中学到了什么？
  - [ ] 我如何改进产品决策流程？
  - [ ] 是否有我应该更新的产品知识？
  - [ ] 我的产品管理技能在哪些方面最需要改进？

- [ ] **下一步 (Next Steps)**
  - [ ] 最优先的产品行动项是什么？
  - [ ] 应该进行什么用户研究？
  - [ ] 应该与谁讨论优先级变化？
  - [ ] 应该何时更新路线图？

---

## 检查清单 (Checklist)

在完成这个思维练习后，确认：

- [ ] 进行了全面的产品和用户指标评估
- [ ] 识别了 3-5 个最高优先级的用户问题
- [ ] 识别了 3-5 个改进机会或新功能机会
- [ ] 制定了清晰的功能优先级
- [ ] 制定了交付计划和成功指标
- [ ] 规划了与各利益相关者的沟通
- [ ] 承认了优先级中的不确定性或假设

---

## 实施与跟踪 (Implementation & Tracking)

- **频率 (Frequency)** - 每周进行一次产品指标审查，每月进行一次深度产品分析
- **沟通 (Communication)** - 定期与工程、设计、营销和执行团队分享产品洞见
- **行动 (Action)** - 将产品决策转化为具体的功能或任务
- **追踪 (Tracking)** - 定期监控用户指标、功能采用率和业务指标
