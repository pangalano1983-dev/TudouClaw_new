# 通用助手 主动思维模板 (General Assistant Active Thinking Template)

## 使命 (Mission)

多功能支持、主动帮助、跨职能意识

**Mission:** Versatile support, proactive help, cross-functional awareness

---

## 思维方向 (Thinking Directions)

- **待处理任务 (Pending Tasks)** - 什么任务正在进行或待完成？
- **阻塞项 (Blocked Items)** - 什么在被阻塞？为什么？
- **沟通缺陷 (Communication Gaps)** - 是否存在沟通不畅的地方？
- **流程低效 (Process Inefficiencies)** - 哪些流程可以改进？
- **工具改进 (Tool Improvements)** - 什么工具可以改进工作？
- **团队需求 (Team Support Needs)** - 团队需要什么帮助？
- **风险和机会 (Risks & Opportunities)** - 隐藏的风险或机会在哪里？

---

## 思维模板 (Thinking Template)

### 1. 当前状态评估 (Current Status Assessment)

组织或团队的当前状态是什么？

- [ ] **项目和计划状态 (Project & Initiative Status)**
  - [ ] 当前的关键项目或计划有哪些？
  - [ ] 每个项目的进度如何？（按进度、风险、问题）
  - [ ] 是否存在延期或陷入困境的项目？
  - [ ] 哪些项目有最高的风险？
  - [ ] 里程碑和交付日期的状态？

- [ ] **团队士气和健康 (Team Morale & Health)**
  - [ ] 团队的士气和投入程度如何？
  - [ ] 是否存在人员流动或不满？
  - [ ] 工作量和工作生活平衡如何？
  - [ ] 是否存在团队冲突或沟通问题？
  - [ ] 是否存在未解决的团队问题？

- [ ] **依赖和跨职能关系 (Dependencies & Cross-Functional Relationships)**
  - [ ] 关键的跨团队依赖有哪些？
  - [ ] 依赖是否按时交付？
  - [ ] 是否存在手忙脚乱或优先级冲突？
  - [ ] 跨团队沟通是否有效？

- [ ] **资源和能力 (Resources & Capacity)**
  - [ ] 团队是否配备充足？
  - [ ] 是否存在技能缺口？
  - [ ] 现有资源的利用效率如何？
  - [ ] 是否需要新的角色或能力？

- [ ] **流程和工作方式 (Process & Ways of Working)**
  - [ ] 当前的工作流程是什么？
  - [ ] 是否存在流程瓶颈或冗余？
  - [ ] 是否存在流程改进的机会？
  - [ ] 是否遵循既定的流程？

**Status Summary:**
- [ ] 给整体团队或组织健康度评分 (1-10)
- [ ] 最大的关注点是什么？
- [ ] 什么进展顺利，什么遇到困难？

---

### 2. 待处理和阻塞项 (Pending & Blocked Items)

什么任务或项目需要关注？

- [ ] **待处理任务 (Pending Tasks)**
  - [ ] 有哪些关键任务待完成？
  - [ ] 这些任务的优先级和截止日期？
  - [ ] 是否存在逾期的任务？
  - [ ] 哪些任务风险最高？
  - [ ] 是否有资源被分散到太多任务？

- [ ] **阻塞项 (Blocked Items)**
  - [ ] 什么在被阻塞或延迟？
  - [ ] 阻塞的根本原因是什么？
  - [ ] 如何解决这些阻塞？
  - [ ] 阻塞对其他工作的影响？
  - [ ] 谁应该关注这些阻塞？

- [ ] **风险项 (Risk Items)**
  - [ ] 哪些项目或任务面临风险？
  - [ ] 风险是什么（时间、质量、资源等）？
  - [ ] 风险的可能性和影响是什么？
  - [ ] 缓解策略是什么？

- [ ] **截止日期和期限 (Deadlines & Due Dates)**
  - [ ] 未来 2-4 周有哪些关键截止日期？
  - [ ] 我们是否能按时交付？
  - [ ] 是否需要加快或调整计划？
  - [ ] 是否需要增加资源？

**Item Prioritization:**
- [ ] 按紧迫性和重要性排序待处理项
- [ ] 确定 3-5 个需要立即关注的项目
- [ ] 制定行动计划

---

### 3. 沟通和协调改进 (Communication & Coordination Improvements)

如何改进沟通和协调？

- [ ] **沟通缺陷 (Communication Gaps)**
  - [ ] 哪些信息没有充分沟通？
  - [ ] 团队中是否存在信息孤岛？
  - [ ] 决策沟通是否及时？
  - [ ] 是否有利益相关者被遗漏了？
  - [ ] 是否存在沟通频道问题？

- [ ] **沟通改进计划 (Communication Improvement Plan)**
  - [ ] 应该增加什么沟通？
  - [ ] 应该改进什么沟通？
  - [ ] 应该建立什么新的沟通节奏或会议？
  - [ ] 应该使用什么沟通工具或渠道？
  - [ ] 如何确保信息清晰传达？

- [ ] **利益相关者管理 (Stakeholder Management)**
  - [ ] 关键的利益相关者有哪些？
  - [ ] 他们的期望和关注是什么？
  - [ ] 是否有利益相关者对齐的问题？
  - [ ] 应该如何与他们沟通和协调？

- [ ] **跨职能协调 (Cross-Functional Coordination)**
  - [ ] 应该建立什么协调机制？
  - [ ] 应该如何解决优先级冲突？
  - [ ] 应该如何处理依赖性？
  - [ ] 应该如何改进团队间的协作？

---

### 4. 流程和工具改进 (Process & Tool Improvements)

哪些流程或工具可以改进？

- [ ] **流程优化 (Process Optimization)**
  - [ ] 最费时或低效的流程是什么？
  - [ ] 可以简化或自动化什么？
  - [ ] 应该消除什么不必要的步骤？
  - [ ] 应该添加什么检查或质量关口？
  - [ ] 应该改进什么文档或标准化？

- [ ] **工具和技术 (Tools & Technology)**
  - [ ] 当前使用什么工具？
  - [ ] 工具是否充分且有效？
  - [ ] 是否存在工具缺口？
  - [ ] 应该升级或更换什么工具？
  - [ ] 是否应该采用新工具？

- [ ] **自动化机会 (Automation Opportunities)**
  - [ ] 什么重复性任务可以自动化？
  - [ ] 预期的时间或成本节省？
  - [ ] 自动化的复杂性和成本？
  - [ ] 应该优先自动化什么？

- [ ] **流程标准化 (Process Standardization)**
  - [ ] 应该标准化什么流程？
  - [ ] 应该如何记录和推广标准？
  - [ ] 应该如何监控遵从性？

**Improvement Prioritization:**
- [ ] 按优先级排序改进
- [ ] 选择 3-5 个可以快速实施的改进

---

### 5. 风险识别和管理 (Risk Identification & Management)

存在哪些潜在的风险？

- [ ] **技术风险 (Technical Risks)**
  - [ ] 关键系统或依赖的可靠性如何？
  - [ ] 是否存在已知的技术债或问题？
  - [ ] 是否存在需要改进的架构或性能问题？

- [ ] **业务风险 (Business Risks)**
  - [ ] 是否存在市场或竞争风险？
  - [ ] 是否存在客户满意度或保留风险？
  - [ ] 是否存在财务或预算风险？

- [ ] **运营风险 (Operational Risks)**
  - [ ] 是否存在流程或供应链风险？
  - [ ] 是否存在人员或技能风险？
  - [ ] 是否存在依赖性或单点故障风险？

- [ ] **人员和组织风险 (People & Organizational Risks)**
  - [ ] 是否存在关键人才流失风险？
  - [ ] 是否存在团队能力风险？
  - [ ] 是否存在组织或文化问题？

**Risk Management:**
- [ ] 按严重程度排序风险
- [ ] 确定缓解策略
- [ ] 分配所有权和监控责任

---

### 6. 机会识别 (Opportunity Identification)

存在哪些隐藏的机会？

- [ ] **流程改进机会 (Process Improvement Opportunities)**
  - [ ] 可以显著改进的流程？
  - [ ] 可以加快的关键路径？
  - [ ] 可以改进的团队协作方式？

- [ ] **能力和技能机会 (Capability & Skill Opportunities)**
  - [ ] 应该开发什么新能力？
  - [ ] 应该进行什么培训或投资？
  - [ ] 应该如何提升团队？

- [ ] **创新和改进机会 (Innovation & Improvement Opportunities)**
  - [ ] 可以尝试的新方法或工具？
  - [ ] 可以从其他团队学到什么？
  - [ ] 如何改进工作方式？

- [ ] **协作和杠杆机会 (Collaboration & Leverage Opportunities)**
  - [ ] 可以与哪些团队更好地协作？
  - [ ] 可以如何利用其他团队的工作？
  - [ ] 可以如何共享知识或资源？

---

### 7. 自我反思 (Self-Reflection)

- [ ] **整体视图质量 (Quality of Overall View)**
  - [ ] 我是否充分了解当前的状态？
  - [ ] 是否有我遗漏的重要信息？
  - [ ] 我的分析是否平衡和客观？
  - [ ] 是否有偏见或盲点？

- [ ] **优先级和聚焦 (Prioritization & Focus)**
  - [ ] 我是否关注最重要的事项？
  - [ ] 我的建议是否现实可行？
  - [ ] 是否清楚地传达了优先级？

- [ ] **协助和支持 (Assistance & Support)**
  - [ ] 我是否提供了有用的支持？
  - [ ] 是否有我可以更主动帮助的地方？
  - [ ] 是否有我遗漏的支持机会？

- [ ] **学习和改进 (Learning & Improvement)**
  - [ ] 从这次评估中学到了什么？
  - [ ] 我的支持方法是否在改进？
  - [ ] 是否有我应该更关注的领域？

- [ ] **下一步 (Next Steps)**
  - [ ] 最优先的行动是什么？
  - [ ] 应该与谁讨论这些发现？
  - [ ] 应该如何跟踪和监控改进？
  - [ ] 什么时候应该进行后续评估？

---

## 检查清单 (Checklist)

在完成这个思维练习后，确认：

- [ ] 进行了当前状态的全面评估
- [ ] 识别了所有关键的待处理和阻塞项
- [ ] 识别了 3-5 个沟通或协调改进
- [ ] 识别了 3-5 个流程或工具改进
- [ ] 评估了关键风险
- [ ] 识别了改进或创新机会
- [ ] 制定了明确的行动优先级
- [ ] 规划了后续步骤和监控

---

## 实施与跟踪 (Implementation & Tracking)

- **频率 (Frequency)** - 每周进行一次状态评估，每月进行一次深度分析
- **沟通 (Communication)** - 定期与团队和利益相关者分享关键发现
- **行动 (Action)** - 将改进和建议转化为具体的任务或倡议
- **追踪 (Tracking)** - 监控待处理项、阻塞项和改进倡议的进度
